# Design Documentation — Methods

Design-time reference documents for the gnomon epistemic framework. These files
describe the rationale, structure, and specification of the research organization
system. They inform the design of the governance content in
`src/gnomon/data/` but are not loaded at runtime.

## Components

| Document | Scope |
| --- | --- |
| [plan-agents-humain-generation](plan-agents-humain-generation.md) | Master plan and problem diagnosis |
| [workspace-architecture](workspace-architecture.md) | Directory layout and file organization |
| [formal-contracts](formal-contracts.md) | Per-note inferential specifications |
| [note-types](note-types.md) | Per-type structural constraints |
| [procedural-workflows](procedural-workflows.md) | Staged pipelines and pass model |
| [registries](registries.md) | Machine-readable project state |
| [migration-reasoning-graph](migration-reasoning-graph.md) | Migration strategy and reasoning graph |
| [plan-revisions](plan-revisions.md) | Plan revision history |

Quality validation tooling (audits, prompt-writing methods) are documented in
[eutaxis](https://github.com/esther-poniatowski/eutaxis).
